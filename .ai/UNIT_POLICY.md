# 仲間ユニット追加ポリシー

## 目的
- ユニット追加時に「強すぎる」「使い道がない」「実装漏れ」を防ぎ、短時間で安全にリリースする。

## 設計原則
- 1ユニット1役割を明確化する（前衛、射程火力、高速突撃、耐久など）。
- `cost` は性能の総量に比例させる。低コストは尖らせず、高コストは欠点も残す。
- `atkInterval` と `atk` はセットで調整し、DPSだけでなく瞬間火力/安定性を分ける。
- `hp` と `speed` は逆相関を基本にする（硬いほど遅く、速いほど脆く）。
- `range` が長いユニットは、耐久か機動力のどちらかを抑える。

## 追加手順（必須）
1. `src/data/unitCatalog.ts` に新規エントリを追加する。
2. `series` と `rarity` を設定し、UI上の分類とガチャ対象に載ることを確認する。
3. 必要なら `src/components/renderers/*` と `src/components/renderers/index.ts` に描画を追加する。
4. `npm.cmd run build` を実行し、型とビルドを確認する。
5. 実プレイで最低1回、以下を確認する。
- 召喚できる
- 射程/攻撃周期が意図通り
- パーティ選択と結果画面で表示崩れがない

## パラメータ目安
- `common`: cost 15-26、明確な弱点を残す
- `rare`: cost 24-35、既存戦術を補強
- `epic`: cost 35-48、構成の核になれる
- `legendary`: cost 48-60、強いが万能にはしない

## 禁止事項
- 既存ユニットの役割を完全に上位互換で潰す値設定
- 追加時に `build` 未実行でコミット
- 仕様説明なしの大幅ナーフ/バフ
