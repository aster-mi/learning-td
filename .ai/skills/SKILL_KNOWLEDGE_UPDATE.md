# SKILL: ナレッジ自動更新

## 目的
作業中に得た知見（成功パターン・失敗パターン・新しい手順）を
Skills とポリシーに反映し、次のエージェントが同じ失敗を繰り返さない状態を維持する。

## いつ実行するか
**毎セッション終了時**、ハンドオフを書く前に以下をチェックする。
該当があれば更新してからハンドオフに含める。

## チェックリスト

### 1. 新しい手順・コマンドを発見したか？
例: 新しい npm script を追加した、新しい検証方法を見つけた
→ 該当する Skill ファイルの「手順」セクションに追記する

### 2. 既存の手順が古くなっていないか？
例: ファイルパスが変わった、コマンドが変わった、ファイル構成が変わった
→ 該当する Skill ファイルを修正する

### 3. 失敗・手戻りがあったか？
例: 汎用テンプレートで作ったらリジェクトされた、品質が悪くてやり直しになった
→ `.ai/lessons/` に失敗パターンを記録する（下記フォーマット）
→ 該当 Skill の「必須方針」や「禁止事項」に反映する

### 4. 新しいカテゴリの作業をしたか？
例: まだ Skill がないタイプの作業をした
→ `.ai/skills/` に新しい Skill ファイルを作成する
→ `README.md` のカタログに追加する

## 失敗パターンの記録

`.ai/lessons/` にファイルを作成する。

ファイル名: `LESSON_<トピック>.md`

```markdown
# <何が起きたか（1行）>

## 状況
- いつ、どの作業で発生したか

## 何が問題だったか
- 具体的な問題の説明

## 根本原因
- なぜそうなったか

## 対策
- どう防ぐか（Skillやポリシーへの反映内容）

## 反映済みファイル
- どのSkill/ポリシーを更新したか
```

## 既存の失敗パターン（参照用）

### expansion.ts の汎用テンプレート問題
- **状況**: Codexが50ユニットを model/gear/aura の組み合わせで一括生成
- **問題**: 全ユニットが同じ体型で見分けがつかない
- **対策**: SKILL_UNIT_ADD.md に「汎用テンプレート禁止」「良い実例」を明記
- **反映**: SKILL_UNIT_ADD.md, UNIT_POLICY.md

### なぞなぞ問題の品質崩壊
- **状況**: 500問を一括生成したら大半が低品質
- **問題**: 意味不明な文、選択肢に解説混入、言葉遊び不成立
- **対策**: SKILL_QUESTION_QC.md に致命/重大/軽微の分類と具体例を追記
- **反映**: SKILL_QUESTION_ADD.md, SKILL_QUESTION_QC.md
