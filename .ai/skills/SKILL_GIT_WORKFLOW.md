# Skill: Git運用

## 目的
安全にコミット・Pushし、作業混線や事故を減らす。

## 基本方針
- 1コミット1意図
- 不要ファイルはコミットしない
- 既存の他者変更は勝手に戻さない

## 手順
1. 変更確認  
  `git status --short`
2. 対象ファイルのみステージ  
  `git add <file>`
3. 明確なメッセージでコミット  
  形式: `<scope>: <short summary>`
4. Push  
  `git push origin main`
5. Push後、必要なら Actions を確認

## 注意点
- `tmp_*` などの生成物を誤って含めない
- 失敗時に `index.lock` が残ったら原因を確認してから再試行
- 破壊的コマンド（`reset --hard` 等）は原則使わない

## 完了条件
- 意図した変更だけが履歴に入っている
- リモートへのPushが成功している
