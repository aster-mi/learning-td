# SKILL: 仲間ユニットバランス調整

## 目的
強すぎる/弱すぎるユニットを是正し、編成の多様性を維持する。

## 対象ファイル
- `src/data/unitCatalog.ts` — ユニットパラメータの定義

## パラメータ一覧
| パラメータ | 意味 | 目安範囲 |
|-----------|------|---------|
| `hp` | 体力 | 50〜350 |
| `atk` | 攻撃力 | 8〜58 |
| `atkInterval` | 攻撃間隔(ms) | 800〜3200 |
| `speed` | 移動速度 | 18〜120 |
| `range` | 攻撃範囲 | 30〜150 |
| `cost` | 召喚コスト | 20〜56 |
| `radius` | 描画サイズ | 13〜21 |

## バランス原則
- `cost` は性能の総量に比例させる
- `hp` と `speed` は逆相関を基本にする（高HPなら低速）
- `range` を伸ばす場合は耐久か機動力を抑える
- 高レアは強くてよいが、完全上位互換は禁止
- 1体で全役割をこなせるユニットは作らない

## レアリティ別の目安
| レアリティ | cost | 総合力イメージ |
|-----------|------|---------------|
| common | 22〜23 | 1つの役割に特化。明確な弱点あり |
| rare | 27〜34 | 2つの強みを持つ。コスト相応 |
| epic | 41〜47 | 高性能だが鈍足 or 低耐久 |
| legendary | 53〜56 | 最高性能。ただし万能禁止 |

## 手順

### 1. 現状の確認
```bash
# 全ユニットのパラメータ一覧（grepで主要値を抜き出す）
grep -oP 'id: "[^"]*".*cost: \d+' src/data/unitCatalog.ts

# 特定シリーズだけ確認
grep -A1 'series: "工学"' src/data/unitCatalog.ts
```

### 2. 問題の分類
- 火力過多: atk × (1000/atkInterval) が同レアの他ユニットより突出
- 耐久過多: hp が同レア帯で突出（特に高speedとの組み合わせ）
- 射程過多: range > 120 かつ hp > 150 など
- 役割不明: 全パラメータが中途半端で採用理由がない

### 3. 調整（1回あたり3〜8体推奨）
`src/data/unitCatalog.ts` の値を直接編集する。

### 4. ビルド確認
```bash
npm run build
```

### 5. 実戦確認
- コストに見合う強さか
- 既存の役割を潰していないか
- 「このユニット要らない」がないか

## 禁止事項
- 1回の調整で全ユニットを一気に変えない
- 高レアでも欠点は残す
- 完全上位互換を作らない

## 完了条件
- 調整後にビルドが通る
- 調整理由をハンドオフに記録している
