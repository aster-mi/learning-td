# Skill: 問題追加・編集

## 目的
`src/data/questions.jsonl` の問題を安全に追加・修正する。

## 対象ファイル
- `src/data/questions.jsonl`
- 必要に応じて `.ai/AGENT_HANDOFF.md`

## 手順
1. 既存IDを確認し、重複しないIDを使う  
  例: `rg -n "\"id\":\"nz" src/data/questions.jsonl`
2. 対象カテゴリ・難易度・サブカテゴリの表記を既存と揃える
3. 1行1JSONの形式を崩さずに追加する
4. 最低チェックを行う
  - `id` が一意
  - `choices` に `answer` が含まれる
  - `choices` は4択
  - 問題文と選択肢の言語・粒度が揃っている
5. 変更内容を handoff に記録する

## 品質ルール
- 問題文は短く自然な日本語
- 選択肢で答えを露骨に示さない
- 解説文字列を選択肢へ混ぜない（例: `○○→△△` の乱用を避ける）
- 同じ意味の選択肢を並べない

## 完了条件
- JSONLが壊れていない
- 問題の意図が第三者に伝わる
- 重複ID・矛盾がない

## NG例
- 1行に複数問題を書く
- 文字化け状態で保存する
- `answer` が `choices` に存在しない
