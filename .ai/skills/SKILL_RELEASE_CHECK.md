# Skill: リリース前チェック

## 目的
公開前の見落としを減らし、問題品質と運用品質を確保する。

## チェック項目
1. データ整合
- JSONL破損なし
- 重複IDなし
- `answer` が `choices` に含まれる

2. 品質
- 問題文が自然
- 選択肢が答えを露骨に示していない
- レベル・カテゴリが妥当

3. 技術
- `npx tsc -b` が通る
- 必要なら `npm run build`（ローカル制約がある場合はCIで確認）

4. 運用
- handoff追記済み（`.ai/AGENT_HANDOFF.md`）
- 重要判断は `.ai/DECISIONS.md` へ記録
- 次アクションを `.ai/TODO.md` に反映

## 完了条件
- チェック項目に未対応がない
- 失敗項目がある場合、理由と次対応が記録されている
