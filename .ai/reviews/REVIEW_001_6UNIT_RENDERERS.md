# REVIEW-001: 6ユニットレンダラー（Codex実装）

レビュー日: 2026-03-21
対象コミット: Codexによる6ユニット追加
レビュアー: Claude

## 総評

全体的にプロンプトの指示に従い、固有シルエットで描画されている。前回の汎用テンプレート問題は解消済み。
ただし、いくつか共通の課題と個別の改善点がある。

**評価: B+（実用レベルだが磨きしろあり）**

---

## 共通の課題

### 1. `col` 引数を使っていないユニットがある
- `drawNotebook`: `void col;` で明示的にスキップ。固定色 `#60a5fa` 系で描画。
- `drawGraphpaper`: `void col;` で明示的にスキップ。固定色 `#a78bfa` 系で描画。
- → `col` はユーザーに見えるユニットカラーなので、ボディのグラデーションに使うべき。固定色だとカタログの `color` と描画が乖離する可能性がある。
- **対策**: `drawProtractor` や `drawBattery` のように `col` ベースのグラデーションにする。

### 2. 歩行アニメーション（`ph` の使い方）
- `drawNotebook`, `drawGraphpaper`, `drawPaintbrush` に **脚がない**。`ph` を bob のオフセットにしか使っていない。
- 他のユニット（pencil, beaker等）はすべて脚 + 歩行アニメーションがある。
- **対策**: 小さな脚を追加し、`ph` で legSwing させる。浮遊型ユニットでない限り脚は必要。

### 3. `battery` と `drone` が工学シリーズの配置場所
- `index.ts` で `battery` と `drone` が「芸術シリーズ」コメントブロック内に置かれている（58-60行目）。
- **対策**: 工学シリーズのコメントブロック内に移動する。

## 個別レビュー

### drawNotebook（ノート騎士）— B+
**良い点:**
- リングバインドが背骨のように見える独自の形状
- ページがマントとして広がる表現
- 罫線がアーマーパターンとして効いている
- ペンを剣にするアイデアが名前に合っている

**改善点:**
- `col` を使っていない（固定色）→ body のグラデーションに `col` を使う
- 脚がない → 小さな脚を追加して歩行アニメを入れる
- ページマントの透明度が低く（0.32）、やや見えにくい → 0.45程度に上げるか輪郭線を追加

### drawProtractor（分度器レンジャー）— A-
**良い点:**
- 半円形のボディが分度器そのもの。シルエットが一目でわかる
- 目盛りの刻みが丁寧（10度刻み + 30度ごとの長目盛り）
- 中心穴の表現もある
- 角度射撃のトレイルエフェクトが動的で良い
- `col` を正しく使い、脚もある

**改善点:**
- 脚のアニメーションが `t` ベース（`Math.sin(t * 3)`）になっている → `ph` を使うべき（他ユニットとタイミングを合わせるため）
- 目のサイズがやや小さく表情が弱い

### drawGraphpaper（方眼スナイパー）— B
**良い点:**
- グリッド線が体に描かれている方眼紙の表現が良い
- スナイパーライフル（丸めた方眼紙）のアイデアが秀逸
- レティクル（照準線）の目がスナイパーらしい

**改善点:**
- `col` を使っていない（固定色 `#f5f3ff`）→ body を `col` ベースに
- **脚がない** → スナイパーなので這い姿勢もありだが、最低限の脚は必要
- ライフルの角度アニメが微妙すぎて（0.06ラジアン）ほぼ静止に見える → もう少し動かすか、スコープのレンズ反射アニメを追加
- body が単なる `roundRect` で少し平凡 → 方眼紙を折ったような角張った形状にするとより個性的

### drawPaintbrush（絵筆ウィザード）— A
**良い点:**
- パレット型のボディが絵筆ウィザードに完璧にマッチ
- 5色の絵の具ブロブが鮮やかで視認性が高い
- 筆が杖になっている構図が良い
- カラフルなマジックスプラッシュのパーティクルアニメーションが綺麗
- キャンバス地のローブの表現もある

**改善点:**
- **脚がない** → ローブの下に脚か、ローブ裾の揺れアニメを入れる
- `globalAlpha` を変更して戻している（1321行）→ `ctx.save()/restore()` で囲む方が安全
- パレットの thumb hole が少し大きすぎる印象

### drawBattery（バッテリー兵）— A-
**良い点:**
- 円筒形の電池ボディが一目でわかる
- +端子のヘルメット、−端子のシューズという発想が良い
- 稲妻マークのパスが丁寧に描かれている
- 充電パルス発光のアニメーションがある
- `col` をボディグラデーションに正しく使用

**改善点:**
- パルス発光が全体を覆うellipseで、ボディが隠れてしまう瞬間がある → パルスは輪郭線 or 上部のみに限定するか、alpha上限を下げる（0.55→0.35程度）
- 脚がなく、代わりに−端子のシューズがあるが、歩行アニメーション（`ph` による足の動き）がない → bob だけでなく、シューズに legSwing を入れる
- 顔が「一文字口」で無表情すぎる → 笑顔 or 真剣な口に変更するとキャラクター性が出る

### drawDrone（ドローン先生）— A
**良い点:**
- クアッドコプターの構造（4アーム + 4ローター）が正確
- プロペラの回転アニメーション（`t * 14` の高速回転 + 楕円で回転感表現）が秀逸
- 浮遊シャドウで接地していない感じが出ている
- 学帽（graduation cap）で「先生」を表現
- カメラレンズの目がラジアルグラデーションで立体的
- LEDインジケーターの点滅アニメーション

**改善点:**
- 浮遊型なので脚なしは正当。ただし `ph` がほぼ使われていない（hover のオフセットのみ）→ 4ローターの回転位相を `ph` で少しずらすと、歩行時に回転が変化して面白い
- 学帽の形がやや平面的 → タッセル（帽子の房）を追加すると「先生感」が増す
